# 1.3 Ключевые проблемы

- **Зависимость от WP:** тема, хуки, шорткоды, админка — всё завязано на WordPress
- **Фрагментация логики:** 16 отдельных плагинов с пересекающимся функционалом
- **Дублирование:** группы «Пользователи» (3 плагина), «Интеграции» (3 плагина), «Админка» (2 плагина)
- **Нет единого API-слоя:** фронтенд обращается к WP через ajax, шорткоды и прямые вызовы
- **Подготовка к миграции не начата:** нет абстракции над БД, нет REST API

# 7. Подготовка к Этапу Б (миграция на новый стек)

Решения в Этапе А, которые упростят будущий переход:

| REST API (JSON) | Node.js-бэкенд будет использовать те же контракты |
| PDO-абстракция с параметризованными запросами | Легче заменить MySQL-драйвер на Postgres |
| Модульная структура `bm-core` | Каждый модуль переносится независимо |
| Фронтенд на чистом JS (без jQuery/WP) | Проще обернуть в React-компоненты |
| Отдельная папка `/api` | Готовый маршрут для Node.js API |
| Миграции БД в коде | Адаптируются под Postgres |


# 8. Риски и рекомендации

| Потеря SEO-позиций при смене URL | Высокая | 301-редиректы со всех старых URL |
| Ошибки при переносе данных из WP-таблиц | Средняя | Скрипты миграции + проверка целостности |
| Расползание сроков (scope creep) | Высокая | Жёсткие границы фаз, MVP-подход |
| Потеря функциональности при удалении плагинов | Средняя | Чеклист функций «до/после» для каждого плагина |
| Недооценка сложности OAuth/платежей | Средняя | Начать с минимальной версии, расширять итеративно |
